April 25, 2019 · WW模型

AVB在PF线以上采样效率的测试

实验背景和目的

在WW模型的PF线以上的区域我们观察到cluster算法给出的cluster非常大。文献中对此的讨论主要有：

  1. 只有在二维Ising模型中SW和Wolff算法对应的逾渗模型的逾渗阈值$p_c$才与Ising模型的临界温度$T_c$对应。在三维中虽然不对应，但是问题不严重。这意味着不是每个推广的SW、Wolff算法都能够对应一个逾渗模型，使得$p_c$与$T_c$对应（具体的探讨并不清楚，这里只是根据文献中的描述进行的推测）。在WW模型中，对应的逾渗模型逾渗阈值非常低，对应一个特征温度$T_0$可能远大于模型的$T_c$，这样逾渗区域$p>p_c$对应温度区域$T<T_0$，存在一个$T_c<T<T_0$的区域发生逾渗（正常应该是$T<T_c\approx T_0$发生逾渗）。

  2. 参考Yuan的文章，在PF线以上的区域有可能存在一些介于F相和P相之间的“相”，Yuan的文章中称之为cluster fluid和percolating fluid。这些“相”的存在必然是和长短程竞争的相互作用有关的。在WW模型中当然也有可能存在。在这个区域，SF等算法必须特别小心，因为这些算法可能表现得很正常，但是可能采样没有达到系统平衡。单从能量或者磁化强度的轨迹上看可能会让人误认为达到平衡，这时候务必使用更精细的参数来讨论平衡问题。如果能够达到平衡，可以考察这个区域内cluster（simple cubic percolation定义下的cluster，并非cluster-flip算法定义的Cluster）的大小与分布。至于其边界，则有可能无法通过热力学量来描述，而是通过percolation参数来描述。

  3. 鉴于第2点，我们希望能够用有效的算法在这个区域内采样，获得平衡统计下的cluster尺寸分布，确定这个区域确实存在这样的问题。所以我们需要比较准确地知道AVBMC相比SF在这个区域内的采样效率。

  4. 至于这个区域出现大的cluster这一点如何影响cluster算法（二者的cluster定义并不相同），可能在后续的研究中会仔细探讨。这是个很有趣的问题。

综上，本实验的目的是表征AVBMC相对SF在“big cluster”区域的采样效率。

可以测定的量除了能量和磁化强度的自关联时间外，更精细的量可以是自旋自关联时间。

此外辅助的手段是：在获取自关联时间后，得到以自关联时间为间隔的一系列snapshot。如果真的达到平衡，这些snapshot应该是独立的样本。因为在这个区域内存在可能性：采样算法被trap到一个局部极小值附近，这样从能量和磁化强度上看似乎是达到了平衡；然而我们真正需要达到平衡的构型却没有达到平衡，所以通过观察这些本该独立的snapshot构型是否真的没有关联，则在自旋关联时间的基础上进一步证实采样的独立性。

自旋自关联时间的计算可能需要较大地重构程序。

这之后能够获得一系列独立的样本，通过这些样本可以计算cluster size distribution以及percolation probability，来尝试确定这个区域的边界。
