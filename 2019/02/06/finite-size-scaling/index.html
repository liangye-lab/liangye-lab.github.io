对PF相变的有限尺度标度计算

实验背景和目的

虽然在上一个实验中，改进的cluster算法(GC)比较成功，但是通过有限尺度体系的热容峰求相变温度仍然不够可靠。更专业的相变温度的计算方法仍然是有限尺度标度(finite size scaling, FSS)。

通过这部分计算，我们将能够以较高的精度得到PF相变的温度数值。

程序版本

实验设计

1. 程序修改

  1. 为了计算WW模型的FSS，首先要求磁矩的高阶矩。对于PF相变，磁矩很容易统计。程序需要增加对磁矩的统计。
  2. 具体计算的时候，考虑好计算所需的步数。

2. $\kappa=0.07$

  1. 先算两个较小体系$N=10^3$和$N=20^3$，温度范围取1.5~3.0，步数取5e4次，看磁矩曲线的形状是否合理，看四阶矩的交点在什么地方。
  2. 在这个交点附近继续算$N=30^3$、$N=40^3$以及$N=50^3$看看是否符合预期。

    数据分析

    $\kappa=0.07$

  1. 首先通过10、20和30三个尺寸下步数为5e4次的粗略计算得出$U_4$的交叉点温度应该在2.2到2.3之间。同时注意到误差特别大，需要增加步数。

  2. 将精度提高到$dT=0.01$，并且将步数提高到1e5次。结果发现直接计算$U_4$的效果并不好。在相变点附近，磁矩轨迹抖动很大，采样看起来不完整。必须通过增加步数来提高精度。

  3. 步数提高到2e6次。从$N=5^3$的$U_4$计算结果来看，首先误差虽然显著减小，但是仍然需要进一步减小；其次整条曲线显出折线形，令人困惑。
    观察$N=10^3$，看到两条$U_4$曲线交在2.198附近。其他地方的皱褶不影响这个判断。

  4. 采用trivial parallel思路，算100个1e5达到算1个1e7的效果。完全可行。$N=5^3$和$N=10^3$几乎是秒出。这两个尺度下误差控制得还可以接受，清楚地看到交点在2.1935。
    然而加上$N=20^3$的线之后发现三条线并不交于一点，三个交点分得很开。
    加上$N=15^3$的线之后发现，后后三条线交于2.205(1)附近。尽管后三条线的误差还是很可观的，但是交点确实给的比较近了。
    但是这里的统计误差不可忽视。改变cutoff后交点的位置也会改变，所以必须进一步提高精度。

结论

从目前的结果来看，Wolff算法在$\kappa=0.07$的时候靠近$T_c$时的采样效率比较差。这个现象出乎意料，因为我们至少认为在PF线上Wolff算法是可以用的。所以按照目前的结果，不能直接求出$U_4$。

下一步

搞清楚为什么Wolff算法在$\kappa=0.07$这样的低阻挫下都会显著降低效率。这需要(1)检查代码；(2)观察在相变点附近算法的表现，是不是产生了大的cluster。
