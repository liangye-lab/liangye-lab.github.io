改进的cluster算法计算低κ区相变温度

实验背景和目的

在Widom-Wheeler (WW)项目中，我们希望计算模型的相变温度。

如果按照Ising模型相变的思路来计算临界温度的话，通常使用Wolff cluster算法并可能配合finite size scaling计算相变温度。如果使用Single Flip (SF)方法的MC，那么本身在一个（接近）二级相变的地方可能有比较强的临界降速(critical slowing down)，而Wolff cluster算法可以很好地避免这个问题（为什么？）。

在WW模型中，也可以构造对应的Wolff Cluster算法，但是这个算法在低温下不是很好用。之前通过粗略的数值计算发现[未归档]，Widom-Wheeler (WW)模型的相变温度估计值都比较低，$T\sim 1$。我们发现对于PF相变，在$\kappa$比较低($\sim0.05$)的时候，相变温度比较高$T_c>2$，Wolff Cluster算法的效果非常好，能够通过计算热容的峰来寻找相变温度（当然也可以使用高阶矩做finite size scaling）[未归档]。然而，对于更高的$\kappa$，相变温度很低，计算效果并不好，体现在可能存在热容峰的温度附近MC难以达到平衡。

通过观察，我们发现，在低温下Wolff cluster算法会将几乎整个系统都加入到cluster中，所以每次翻转的效率并不是很高。虽然三种键对应的加入概率大致合理，但是可能由于超高的连通性（24），每个site被加入到团簇中的概率大大提升，最终整个体系都被加入到团簇中。

为了提高Cluster算法的效率，我们对这一算法做出改进。改进原理参照Frenkel的教科书，简而言之，是牺牲了Acc ratio使得可以在更高温度下构造团簇。为了求温度$T$下的系综，我们在$T’>T$下构造团簇，这样团簇可以比较小；然后按照概率$acc$翻转来接受新构型。当$T’=T$时，$acc=1$，程序退化为Wolff算法；当$T’\to\infty$时，程序退化为SF算法。

由于初步的尝试看到，改进的cluster算法可能在高$\kappa$的区域仍然效果不加，所以本实验主要研究低$\kappa$下的相变温度。

  1. 求$\kappa=0.08\sim0.10$的PF相变温度
  2. 求$\kappa=0.10\sim0.16$的PM相变温度
  3. 观察$\kappa=0.10$以及$\kappa=0.16$：时算法能否找到平衡。
  4. 注意能否发现中间相。

程序版本

WidomWheelerGenCluster, v1.0.0

实验设计

  1. 我们设定一系列的温度倍数$n=T’/T$，来获得给定温度、给定阻挫下的系综，计算平衡后的平均能量和热容。取能够达到平衡的倍数$n$，观察其snapshot是否真的达到目标相。
  2. 在某一$\kappa$下，取能够达到平衡的$n$，计算各个温度下的热容并作图，找峰，确定相变温度。

为了使得平衡更容易找到，我们分别观察从完全无序的构型、从目标构型和从两者的1:1混合构型出发的MC，看混合构型会不会更容易达到平衡。

为了保证平衡收敛，MC轨迹取$5\times10^6$步（每步是一个cluster flip）。

数据分析

1. PF相变

  1. 首先观察能量轨迹：
    (1) 温度在1.0时，基本上所有的$n$（包括$n=0$）都能够在2,000,000步内收敛。存在一个最优的$n$，使得收敛最快。例如，$T=1.0, \kappa=0.07$时$n=1.30$在70,000步左右就能收敛，而$n=0$需要260,000步才能收敛，$n=1.70$在1,800,000才能收敛。对$T=1.0,\kappa=0.09$则是$n=1.40$为最优。
    (2) 温度在2.0时，明显$n=0$即原始的Wolff算法效果更好，收敛速度更快。

  2. 观察snapshot。观察证实确实达到了期望的F相。

  3. 取适宜的n，计算热容。发现取轨迹后半段，得到的热容曲线就很光滑，峰值显著，只有在超低温$T<0.7$左右时才会出现不合理的未平衡的数值。我们认为这个地方不存在相变，这些不合理点是算法失效所致，所以不予深入探讨。

  4. 求相变温度。得到$T=2.2, \kappa=0.07$，$T=1.8, \kappa=0.08$，$T=1.4, \kappa=0.09$，与之前的估计非常接近。当然精度可以进一步提高。

2. PM相变

  1. 首先观察能量轨迹：这里以$\kappa=0.13$为例
    (1) 温度较低时，在$T=1.0$，$n$取$1.50$可以得到很好的结果。当然如果温度降到$0.7$，那么最优$n=1.70$。如果温度进一步降，那么可能需要继续增加$n$。
    (2) 温度较高时，如$T>1.3$，$n$取越小越好。这时候高$n$虽然也在平衡，但是采样效率极低。

  2. 当$\kappa$进一步升高时（$T=0.15$），可以感受到相变温度在降低($T\sim0.7$)，但是在现在的$n$值下最好的是$n=2.5$，在$T=0.6$仍然没有达到平衡（尽管可以看到M相开始形成）。

  3. 观察snapshot，$\kappa<0.15$时，低温都能够降到$\langle 2\rangle$上；但是高$\kappa$尚未平衡。值得注意的是，$n=2.5, \kappa=0.16, T=0.6$从混合构型开始的MC最终落在了一个“稳定”的相上，这个相呈现出蜂窝状调制结构和$\langle 2\rangle$并存的结构。MC能量显示能够在这个状态上稳定，不知道还有没有更低能量的构型，假如跑时间再长一点能够落下去。

3. 多相点

$\kappa=1/10$

从能量曲线上看，完全达不到平衡。从snapshot上看，系统进入了一个比较靠近F相的状态，内部spin形成大块，可能存在比较高的截面自由能垒。因此无法确定相变状况。

$\kappa=1/6$

从能量曲线上看，难以达到平衡。但是$n=2.5, \kappa=0.16, T=0.6$从混合构型开始的MC最终落在了一个“稳定”的相上。从snapshot上看，这个相呈现出蜂窝状调制结构和$\langle 2\rangle$并存的结构。

由于不确定是否可重复，现在不能对此下结论。因此无法完全确定相变状况。

结论

  1. 算法在$\kappa<0.15$表现很出色，不仅给出了清晰的热容峰，而且最终能够在低温区自发地落回目标相。但是在低温时需要运行约30个小时(dcc)才能达到平衡。

  2. 由于温度取点只是初步的，相变温度的精度不够高(HALP)，这一点可以通过在相变点附近增加取点解决。

  3. 对于可能存在的复杂区域$\kappa\sim 0.1$和$\kappa\sim 1.6$，目前无法得出任何可靠结论，因为在这些点没有得到清晰的收敛结果，只能说是一个估计。

下一步

  1. 可以增加取点，求出更精确的相变温度。

  2. 可以初步尝试更高的$\kappa$。

  3. 关于$\kappa=1/10$和$\kappa=1/6$处的相图结构，还需要想更多的办法。
